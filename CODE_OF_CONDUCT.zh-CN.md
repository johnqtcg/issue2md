# 社区行为准则 (Code of Conduct)

本项目致力于为所有参与者提供一个友好、安全、包容的协作环境。无论你的经验水平、身份背景、表达方式或观点如何，都应被平等尊重。

## 1. 我们的承诺 (Our Pledge)

在本仓库的所有协作场景中，我们承诺：

- 以尊重、耐心、建设性的方式沟通。
- 聚焦技术问题本身，避免人身攻击。
- 接受合理反馈，并在分歧中保持专业。
- 主动保护他人隐私与安全。

## 2. 期望行为 (Expected Behavior)

- 使用清晰、礼貌、可执行的技术表达。
- 对 PR/Issue/Review 给出基于事实的反馈。
- 在讨论中提供上下文、证据和可复现信息。
- 对新人和不同观点保持包容。

## 3. 不可接受行为 (Unacceptable Behavior)

- 骚扰、侮辱、歧视、威胁或任何人身攻击。
- 恶意挑衅、刷屏、持续性破坏讨论秩序。
- 未经许可公开他人隐私、凭据或敏感信息。
- 明知有害仍传播漏洞利用细节、恶意代码或误导性信息。
- 任何违反 GitHub 社区规范和当地法律法规的行为。

## 4. 适用范围 (Scope)

本准则适用于以下场景：

- 仓库内的 Issue、PR、Review、Discussion、Commit comments。
- 与本项目相关的公开技术沟通渠道。
- 代表本项目进行的公开交流行为。

## 5. 举报方式 (Reporting)

如果你遭遇或观察到违规行为，请尽快报告：

1. 一般协作行为问题：在仓库中提交 Issue（建议标题前缀：`[CoC]`），描述事实与时间线。
2. 涉及隐私、安全或不适合公开的信息：请按 [SECURITY.md](SECURITY.md) 使用私密上报通道，不要公开披露敏感细节。

报告建议包含：

- 发生时间与位置（链接）。
- 相关上下文和客观事实。
- 你期望的处理方式（如删除评论、警告、限制互动等）。

## 6. 处理流程 (Enforcement Process)

维护者会基于事实进行评估，并尽量在 3 个工作日内给出初步响应。处理原则：

- 公平：基于证据，不做预设立场。
- 最小必要：仅披露处理所需信息，保护举报人隐私。
- 渐进式处置：从提醒、警告到限制互动，严重情况可直接移除协作权限。

## 7. 可能的处置措施 (Consequences)

根据违规严重程度，维护者可采取：

- 口头/书面提醒；
- 删除或隐藏不当内容；
- 暂时限制评论或协作权限；
- 永久禁止参与本项目协作。

## 8. 参考与声明 (Attribution)
