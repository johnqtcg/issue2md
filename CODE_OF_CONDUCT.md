# Code of Conduct

Language:
- English (primary): `CODE_OF_CONDUCT.md`
- Chinese: [CODE_OF_CONDUCT.zh-CN.md](CODE_OF_CONDUCT.zh-CN.md)

## 1. Our Pledge

We are committed to making participation in this project a respectful, safe, and inclusive experience for everyone.

In all project spaces, we commit to:
- Communicate respectfully and constructively.
- Focus on technical issues, not personal attacks.
- Accept feedback professionally.
- Protect privacy and security.

## 2. Expected Behavior

- Use clear, respectful, and actionable communication.
- Give evidence-based feedback in Issue/PR/Review discussions.
- Provide context and reproducible details when reporting problems.
- Be welcoming to newcomers and different viewpoints.

## 3. Unacceptable Behavior

- Harassment, insults, discrimination, threats, or personal attacks.
- Deliberate disruption, trolling, or repeated low-signal spam.
- Sharing private information, credentials, or sensitive data without permission.
- Publishing exploit details or malicious payloads irresponsibly.
- Any behavior that violates GitHub policies or applicable laws.

## 4. Scope

This Code of Conduct applies to:
- All repository spaces: Issue, PR, Review, Discussion, comments.
- Public communication channels directly related to this project.
- Any project-representative public interactions.

## 5. Reporting

If you experience or witness unacceptable behavior:

1. For general conduct issues, open an Issue with prefix `[CoC]` and provide a factual timeline.
2. For sensitive/security-related incidents, use the private reporting path in [SECURITY.md](SECURITY.md) and avoid public disclosure.

Please include:
- Where and when it happened (links if available)
- Factual context
- Preferred handling outcome

## 6. Enforcement Process

Maintainers will review reports based on evidence and aim to provide an initial response within 3 business days.

Enforcement principles:
- Fairness: evidence-based decisions
- Privacy: disclose only what is necessary
- Proportionality: warnings first when appropriate; stronger actions for severe/repeated violations

## 7. Possible Consequences

Depending on severity, maintainers may:
- Issue a warning
- Remove or hide inappropriate content
- Temporarily limit participation
- Permanently ban participation

## 8. Attribution

This policy is informed by Contributor Covenant practices and adapted for this project's collaboration workflow.
